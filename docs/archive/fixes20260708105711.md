# Comprehensive Bug Fix Plan (2026-07-08)

## Deep Sleep Strategy Decision
Given the watch is fully encapsulated with no physical power button, **System ON Sleep** (which FreeRTOS achieves via `suspendLoop()` and its tickless idle mode) is the only viable option. System ON sleep draws slightly more power but keeps the RTC alive, allowing us to pulse the low-battery LED every 30 seconds and wake up instantly via the IMU interrupt or a BLE connection.

## 2. App (`Flutter` / Dart)

### 🚨 Critical Severity
#### 2.1 Missing Background Isolate Migration
*   **Location:** `app/lib/services/foreground_service.dart`
*   **Exact Fix:**
```dart
@override
void onStart(DateTime timestamp, SendPort? sendPort) {
    _bleService = BleService();
    _locationService = LocationService();
    _bleService!.initialize(); // Synchronous
    _locationService!.start();
}
```

#### 2.2 Missing Android 14+ Foreground Service Types
*   **Location:** `app/lib/services/foreground_service.dart` (`startService`)
*   **Exact Fix:** Add `foregroundServiceType: [ForegroundServiceType.location, ForegroundServiceType.connectedDevice],`.

#### 2.4 Loss of location permissions on upgrade failure
*   **Location:** `app/lib/services/location_service.dart`
*   **Exact Fix:** Check `upgradedPermission != LocationPermission.denied && upgradedPermission != LocationPermission.deniedForever`.

#### 2.5 Unbounded Infinite Reconnects Loop
*   **Location:** `app/lib/services/ble_service.dart`
*   **Exact Fix:** Cap attempt counter to 20, and just `return false;` if exceeded instead of infinite loop.

#### 2.7 Negative Jitter Crash
*   **Location:** `app/lib/services/bearing_calculator.dart`
*   **Exact Fix:** Use `math.max(1, remoteMaxS - remoteMinS + 1)` instead of `math.max` (wait, the fix is to add the 1 as a lower bound: `math.max(1, ...)`).

#### 2.8 BLE Concurrency Bug (Bypassed Write Queue)
*   **Location:** `app/lib/services/ble_service.dart` (`sendCalibCmd`)
*   **Exact Fix:** Await `_enqueueWrite(() => _ble.writeCharacteristicWithResponse(characteristic, value: cmd));`.

### ⚠️ High Severity
#### 2.9 Unhandled Exceptions
*   **Location:** `app/lib/screens/settings_screen.dart` (1 call site), `app/lib/screens/wrist_flick_calibration_screen.dart` (2 call sites), `app/lib/screens/wake_calibration_screen.dart` (1 call site).
*   **Exact Fix:** Wrap Config DB saves (`saveConfig`) and BLE writes (`writeConfig`) in separate `try-catch` blocks across all 4 call sites. Show distinct `SnackBar` messages for DB failures vs. BLE sync failures so users/support can tell which layer broke.

#### 2.10 Unsafe BLE Setup Sequence
*   **Location:** `app/lib/services/ble_service.dart` (`_onConnected`)
*   **Exact Fix:** Add `await Future.delayed(const Duration(milliseconds: 150));` between characteristic discovery steps.

#### 2.11 Supabase Realtime Subscription Leak
*   **Location:** `app/lib/services/sync_service.dart`
*   **Exact Fix:** Add `_isSubscribing` boolean lock to prevent duplicate channel creation.

#### 2.12 Out-of-Order Database Updates
*   **Location:** Supabase Database (Postgres)
*   **Exact Fix:** Create a `BEFORE UPDATE` Postgres trigger to drop or reject incoming records that have an older timestamp or sequence than the currently stored record. This guarantees single-source-of-truth ordering even across multiple writers or offline sync replays.

#### 2.13 Reconnect Attempt Counter Never Resets
*   **Location:** `app/lib/services/ble_service.dart` (`connectToDevice`)
*   **Exact Fix:** Set `_reconnectAttempt = 0;` unconditionally on manual connect.

#### 2.14 syncTime Infinite Retry Loop
*   **Location:** `app/lib/services/ble_service.dart` (`syncTime`)
*   **Exact Fix:** `if (attempt > 3) return;`

### 🟡 Medium Severity
#### 2.15 gpsIntervalRemoteMaxS Slider Lockout
*   **Location:** `app/lib/screens/settings_screen.dart`
*   **Exact Fix:** Implement a `RangeSlider`.

#### 2.17 Resource Leak TextEditingController
*   **Location:** `app/lib/screens/settings_screen.dart` (`_showColorPicker`)
*   **Exact Fix:** Dispose controller after dialog closes.

#### 2.18 State Leak: Explicit disconnect() leaves _connectedDeviceId cached
*   **Location:** `app/lib/services/ble_service.dart` (`disconnectFromDevice`)
*   **Exact Fix:** Set `_connectedDeviceId = null` when the user explicitly disconnects. Do NOT clear it on accidental connection loss (e.g. out of range), so that the auto-reconnect logic still attempts to target the correct device.
