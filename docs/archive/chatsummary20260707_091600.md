# Chat Summary - 2026-07-07

## Context
The user reported multiple severe bugs in the Nexus Halo project:
1. The watch completely died/locked up after entering low power mode, requiring a hard reset.
2. LEDs were seemingly acting strange in clock mode (reporting 5 LEDs on).
3. The app was failing to save configurations, reverting to previous ones.
4. The manual calibration screen was using the "wake-up interrupt event" instead of real-time `mg` data to calibrate the slider.

## Investigation & Fixes Applied
We utilized a multi-agent teamwork strategy to systematically investigate and fix each issue, ensuring rigorous code reviews:

1. **System OFF Sleep Bug (`power.cpp`)**: 
   - Found that `sd_power_system_off()` was being used, which cuts the USB peripheral and prevents standard wake-up without SENSE pins. 
   - Reverted to `System ON` sleep via `sd_app_evt_wait()` and `__WFE()`, preserving memory and interrupt behavior at a negligible 1-2uA cost.

2. **I2C Deadlock Bug (`nexus_halo.ino`)**:
   - The Mbed OS workaround disabling `NRF_TWIM1`/`NRF_TWI1` registers was conflicting with the internal LSM6DS3 IMU wired to `Wire1`, causing a hard fault on wake. Removed the aggressive register disabling for `Wire1`.

3. **Haptic Controller Blocking Loop (`haptic.cpp`)**:
   - The `playPattern` function used `delay()`, starving the BLE event loop. Refactored to integrate with the pre-existing non-blocking `update()` logic.

4. **Radar Timeout Ping-Pong (`state_machine.cpp`)**:
   - Rapid switching between Radar and Distance modes caused a ping-pong infinite loop timeout, draining the battery. Fixed the exit state explicitly calling `haptic.stopMotor()` to prevent stuck vibrations, and fixed timeouts to safely fall back to the base Clock modes.

5. **Multi-LED Watch Face**:
   - Investigated the 5 LEDs issue. The clock code precisely mathematically restricts itself to 3 LEDs. The extra LEDs were actually due to battery/error overlays activating properly. No fractional rendering bugs existed.

6. **App Configurations & UI**:
   - Verified that the `partner_repository.dart` correctly throws errors via `rethrow` to the UI.
   - Verified `settings_screen.dart` catches the error and surfaces it in a red `SnackBar`.
   - The `wakeThreshold` max was increased to 63, and `gyroThreshold` to 2000.
   - The `wake_calibration_screen.dart` was updated to subscribe to real-time IMU streaming via `ble.startImuStream()` to display live `mg` and `dps` values, vastly improving the calibration experience.

## Status
All fixes have been deployed and reviewed by subagents. A new release APK was compiled.
