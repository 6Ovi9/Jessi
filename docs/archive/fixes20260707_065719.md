# Nexus Halo Fixes - 2026-07-07

This file contains the proposed code fixes for the bugs identified in `bugs20260707_065719.md`.

## 1. System OFF Deep Sleep Lockup
**File:** `nexus_halo/power.cpp`

**Fix Description:** We must revert from `System OFF` mode (which cuts power and USB, preventing wake up) to `System ON` mode, which preserves state and allows normal interrupt wake-ups. The expected battery cost is negligible (2-5uA).

```cpp
void PowerManager::enterDeepSleep() {
    Serial.println(F("[POWER] Entering Deep Sleep (System ON)..."));
    delay(50); // Flush serial

    // 1) Set sleep flags
    is_sleeping = true;
    
    // 2) Cut power to external peripherals (if applicable)
    // digitalWrite(PERIPHERAL_POWER_PIN, LOW);
    
    // 3) Suspend BLE advertising to save power
    ble_handler.suspend();

    // 4) Sleep loop - Wait for Interrupt (WFI/WFE)
    // Note: This blocking loop means BLE events handled by the SoftDevice
    // will not reach the application layer (e.g. ble_handler.update()) 
    // until a physical wakeup occurs, which is intended.
    while (is_sleeping) {
        uint8_t sd_enabled;
        sd_softdevice_is_enabled(&sd_enabled);
        if (sd_enabled) {
            // Use SoftDevice API for sleep if BLE is active
            sd_app_evt_wait();
        } else {
            // Use bare-metal WFE if SoftDevice is off
            __WFE();
            __SEV();
            __WFE();
        }
    }
    
    // 5) Wake up routine
    Serial.println(F("[POWER] Woke up from Deep Sleep"));
    ble_handler.resume();
}
```

## 2. I2C/TWI Internal Bus Deadlock
**File:** `nexus_halo/nexus_halo.ino`

**Fix Description:** The workaround manually disabling `NRF_TWIM1` and `NRF_TWI1` conflicts with `Wire1` which is used by the internal LSM6DS3 IMU. We remove those lines.

```cpp
// Inside setup()

    // --- I2C BUS INIT ---
    // The Mbed OS core has a known bug where Wire.end() doesn't properly 
    // release the TWI registers, breaking future I2C initialization.
    // We apply a hardware-level workaround for Wire (TWIM0).
    Wire.begin();
    Wire.end(); // Release hardware TWI
    
    // Mbed OS Wire.end() bug workaround: Disable NRF peripherals manually
    if(NRF_TWIM0->ENABLE) NRF_TWIM0->ENABLE = 0;
    if(NRF_TWI0->ENABLE) NRF_TWI0->ENABLE = 0;
    NRF_TWIM0->PSEL.SCL = 0xFFFFFFFF; // Disconnect TWI hardware pins
    NRF_TWIM0->PSEL.SDA = 0xFFFFFFFF;
    
    // NOTE: We specifically do NOT touch TWIM1 / TWI1 here, 
    // because Wire1 is used by the internal LSM6DS3 IMU and disabling 
    // it will cause a hard fault on the Seeed XIAO nRF52840 Sense.

    // Initialize regular I2C now that the bus is clean
    Wire.begin();
    Wire.setClock(400000);
```

## 3. Haptic Controller Event Loop Starvation
**File:** `nexus_halo/haptic.cpp`

**Fix Description:** `playPattern` uses blocking `delay()` calls. We rewrite it to simply set the state and let the *already existing* non-blocking `update()` loop handle the timing. 

```cpp
void HapticController::playPattern(HapticPattern pattern) {
    stopMotor();
    const VibrationStep* steps = _getPattern(pattern);
    if (!steps) return;
    
    current_pattern = pattern;
    pattern_start_ms = millis();
    pattern_active = true;
    
    // Note: The main loop's call to haptic.update(millis()) 
    // will safely handle turning the motor on and off.
}
```

## 4. Radar Mode Battery Drain & BLE Spambot
**File:** `nexus_halo/state_machine.cpp`

**Fix Description:** 
1. `STATE_RADAR_MODE` currently times out to `STATE_DEEP_SLEEP`. It should timeout to the clock mode. We cannot use `previous_state` because rapidly swapping between Radar and Distance modes creates an infinite ping-pong timeout loop.
2. The `HapticController` must explicitly be turned off during `_exitState()` to prevent race conditions where a pattern gets "stuck" vibrating during deep sleep.

```cpp
// 1. Fix _exitState
void StateMachine::_exitState() {
  // Cleanup for exiting current state
  extern HapticController haptic;
  haptic.stopMotor(); // Guarantee haptics are off on state transition
}

// 2. Fix update timeout logic
void StateMachine::update(uint32_t stale_now_ms) {
```

## 5. Multi-LED Interpolation Removal
**File:** `nexus_halo/led_controller.cpp`

**Fix Description:** Update `showClock` to ensure only exactly 3 LEDs (Hour, Minute, Second) are active, removing fractional blending.

```cpp
void LEDController::showClock(uint8_t hours, uint8_t minutes, uint8_t seconds) {
    clear();
    
    // Convert 12-hour format
    hours = hours % 12;
    
    // Calculate precise indices (0 to LED_COUNT-1)
    // Note: If physical LEDs are counter-clockwise, invert logic:
    // index = (LED_COUNT - raw_index) % LED_COUNT
    
    uint8_t sec_led = (seconds * LED_COUNT) / 60;
    uint8_t min_led = (minutes * LED_COUNT) / 60;
    
    // Hour snaps to the exact integer hour block (no interpolation)
    uint8_t hour_led = (hours * LED_COUNT) / 12;
    
    // Ensure safe bounds
    sec_led = sec_led % LED_COUNT;
    min_led = min_led % LED_COUNT;
    hour_led = hour_led % LED_COUNT;
    
    // Apply colors from config
    setLEDBrightness(sec_led, runtime_config.getColor(2), 255); // Seconds hand
    setLEDBrightness(min_led, runtime_config.getColor(1), 255); // Minutes hand
    setLEDBrightness(hour_led, runtime_config.getColor(0), 255); // Hours hand
    
    show();
}
```
