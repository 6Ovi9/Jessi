- **File**: `nexus_halo/runtime_config.cpp` (Line 222)
- **Change**:
  ```cpp

  ```
- **Explanation**:

- **Explanation**: 
  Allows PostgREST requests sent via `anonKey` (unauthenticated sessions) to insert and query haptic events and location sync data without 401 Permission Denied errors, strictly validated for User A and User B.

### Fix 3.1: GATT Sequential Write Queue in Background Engine
- **File**: `app/lib/services/background_engine.dart` (New helper class & execution wrapper)
- **Change**:
  ```dart
  final List<Future<void> Function()> _gattWriteQueue = [];
  bool _isProcessingGattQueue = false;

  Future<void> _enqueueGattWrite(Future<void> Function() writeTask) async {
    _gattWriteQueue.add(writeTask);
    if (!_isProcessingGattQueue) {
      _isProcessingGattQueue = true;
      while (_gattWriteQueue.isNotEmpty) {
        final task = _gattWriteQueue.removeAt(0);
        try {
          await task();
        } catch (e) {
          debugPrint('[BG] GATT Queue Write Error: $e');
        }
        await Future.delayed(const Duration(milliseconds: 50));
      }
      _isProcessingGattQueue = false;
    }
  }
  ```
- **Explanation**: 
  Serializes all BLE characteristic writes (`write_config`, `write_haptic`, `write_bearing`, `write_distance`) into a single FIFO execution queue, preventing Android GATT status 133 concurrent write crashes.
