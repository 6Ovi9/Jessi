# Bug Hunt Report: 2026-07-08

## Agent: Bug Hunter (app)
**Status:** Completed

### 1. Critical: Loss of existing location permissions on upgrade failure
**Location:** `app/lib/services/location_service.dart`
**Issue:** If the user has `whileInUse` permission, the app tries to upgrade to `always`. If denied, it overwrites the existing permission status with `denied`, breaking location tracking entirely despite having sufficient background foreground service permission.

### 2. Logic Bug: `gpsIntervalRemoteMaxS` slider lockout
**Location:** `app/lib/screens/settings_screen.dart`
**Issue:** Only the minimum interval can be adjusted. If the user pushes the minimum interval up, it permanently pushes the maximum interval up, with no UI mechanism to ever bring the maximum interval back down.

### 3. Architecture/Logic: Unbounded Infinite Reconnects Loop Drain
**Location:** `app/lib/services/ble_service.dart`
**Issue:** `_maxReconnectAttempts` is `-1`. The app will infinitely attempt to reconnect with an active BLE scan every 5 minutes forever, draining the phone battery over several days.

### 4. Logic Bug: `syncTime` Infinite Retry Loop
**Location:** `app/lib/services/ble_service.dart`, `syncTime()`
**Issue:** An infinite exponential backoff loop occurs if the time sync characteristic fails, waking up the phone every 15 minutes endlessly for a non-critical feature.

## Agent: Bug Hunter (Architecture/UI)
**Status:** Completed

### 1. 🚨 Critical Battery/Architecture Flaw: Silent Death in DEEP_SLEEP
**Location:** `nexus_halo.ino`, `handleStateDeepSleep`
**Issue:** The watch is supposed to pulse the LED every 30 seconds at critical battery. Instead, it calls `power_manager.enterDeepSleep()`, triggering `suspendLoop()` which completely halts the CPU, preventing the low-battery pulse and allowing the watch to drain silently.

### 2. 🚨 UI/UX Misalignment: TX Feedback Vibrates Against Spec
**Location:** `nexus_halo.ino`, `handleStateHapticTX()` vs `README.md`
**Issue:** To save massive battery draw, the spec requires no motor vibration on TX confirmation, only LED flashes. However, the code calls `haptic.playPattern()`.

### 3. 🚨 Severe Code-Documentation Misalignment: Haptic Gestures
**Location:** `nexus_halo.ino` vs `README.md`
**Issue:** The README claims "Double Flick" sends a haptic touch, but the code requires a "Triple Flick" (`GESTURE_TAP_TRIPLE`).

## Agent: Bug Hunter (backend)
**Status:** Completed

### 1. [Backend] RLS Blocks Database Maintenance (Silent Failure) & Search Path Hijacking
**Location:** `backend/init-db.sql`, `cleanup_old_haptic_events()`
**Issue:** The table has RLS enabled but omits a `DELETE` policy. Because the cleanup function inherits caller policies, the `DELETE` is silently blocked. It requires `SECURITY DEFINER`, which also introduces a `search_path` hijacking vulnerability if not explicitly scoped.

### 2. [Backend] Supabase Studio Networking Misconfiguration
**Location:** `backend/docker-compose.yml`
**Issue:** The `studio` container bypasses Nginx (`http://rest:3000`), breaking auth and realtime websockets.

### 3. [App & Backend] Brittle JWT Documentation / Hardcoded App Token
**Location:** `backend/README.md` and `app/lib/main.dart`
**Issue:** `supabaseAnonKey` is hardcoded. If the user changes `JWT_SECRET` per the README, the app breaks.

## Agent: Bug Hunter (nexus_halo)
**Status:** Completed

### 1. OTA Migration Memory Corruption (`runtime_config.h` & `runtime_config.cpp`)
**Issue:** `RuntimeConfig` is not `__attribute__((packed))`. Struct size changes between versions cause `memcpy` to misalign offsets and completely corrupt user settings on OTA update.

### 2. IMU Register Corruption Bug (`nexus_halo/power.cpp`)
**Issue:** In `setupIMUForRiseToWake()`, shifting `(IMU_WAKE_UP_DUR & 0x03) << 6` is incorrect. The field is bits [6:5], so `<< 5` is mathematically required to prevent corrupting adjacent registers.

### 3. Checksum Padding Risk (`compass.cpp`)
**Issue:** `CompassCalibData` suffers from compiler padding. It must be packed.

### 4. Clock Time Underflow Risk (`nexus_halo/nexus_halo.ino`)
**Issue:** `uint32_t total_secs = (uint32_t)(current_unix_ms / 1000ULL) + local_tz;`
If `local_tz < 0` and the watch just rebooted (low `current_unix_ms`), this underflows to a futuristic timestamp, displaying garbage.

## Agent: Bug Hunter (Flutter)
**Status:** Completed

### 1. Architectural Flaw: Missing Background Isolate Migration
**Issue:** BLE and GPS routines still run on the main UI isolate instead of the foreground task isolate. Android will forcefully suspend the UI isolate, breaking execution.

### 2. Fatal Crash: Missing Android 14+ Foreground Service Types
**Issue:** `startService` is missing the `foregroundServiceType` argument, required by Android 14+.

### 3. Potential Exception / Crash: Negative Range in `Random().nextInt`
**Issue:** If `remoteMaxS` < `remoteMinS`, `Random().nextInt` throws a fatal `RangeError`, silently crashing the polling timer.

### 4. State Leak: Explicit `disconnect()` leaves `_connectedDeviceId` cached
**Issue:** `_connectedDeviceId` is never cleared on disconnect.

## Agent: Bug Hunter (Concurrency & Exceptions)
**Status:** Completed

### 1. Flutter App: Supabase Realtime Subscription Leak
**Issue:** Concurrent calls to subscribe leave the first channel orphaned and permanently listening.

### 2. Flutter App: Out-of-Order Database Updates
**Issue:** Rapid location updates can resolve out of order. Needs a Postgres `BEFORE UPDATE` trigger to ensure authoritative ordering.

### 3. Firmware (C++): Unsafe use of `millis()` inside `noInterrupts()`
**Issue:** Calling `millis()` inside `noInterrupts()` prevents updates, yielding stale values.

### 4. Unhandled `Exception` in UI from `repo.saveConfig`
**Issue:** `await repo.saveConfig(cfg)` is called without a `try/catch`. Offline state crashes the event loop.

### 5. Critical BLE Concurrency Bug (Bypassed Write Queue)
**Issue:** `sendCalibCmd` bypasses `_enqueueWrite()`. Concurrent GATT writes crash the native plugin.

### 6. Unsafe BLE Setup Sequence
**Issue:** Setup uses `await Future.delayed` without checking connection state after the delay.
